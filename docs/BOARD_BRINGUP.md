# Board Bring-Up

The selected first board is the TUL PYNQ-Z2:

```text
Device: xc7z020clg400-1
PL clock target: 100 MHz from PS FCLK0
Data path: PS DDR <-> AXI DMA <-> ldpc_axis_decoder_ip
```

The detailed board flow is in `docs/PYNQ_Z2.md`. The audit of the RTL and
stream contract is in `docs/BOARD_READINESS_AUDIT.md`.

## Build

Install Vivado and the TUL PYNQ-Z2 board files, then run:

```sh
make pynq-z2-project
make pynq-z2-synth
make pynq-z2-bitstream
make pynq-z2-overlay
```

The Tcl flow is non-interactive and rebuilds the Vivado project and block
design from source. It prefers the installed PYNQ-Z2 board definition and falls
back to the raw `xc7z020clg400-1` part with an explicit warning.

Generated outputs:

```text
results/pynq_z2/vivado/ccsds_ldpc_pynq_z2/
results/pynq_z2/reports/
results/pynq_z2/overlay/ccsds_ldpc_pynq_z2.bit
results/pynq_z2/overlay/ccsds_ldpc_pynq_z2.hwh
```

## Hardware Test

Copy the entire `results/pynq_z2/overlay/` directory to the PYNQ-Z2 board.
From that directory on the board:

```sh
python3 smoke_test.py
python3 smoke_test.py --random-frames 3
python3 benchmark.py --frames 10
```

The smoke test loads the overlay, verifies `axi_dma_0`, runs the repository
golden model, transfers frames through AXI DMA, and compares status plus
decoded bits.

## DMA Contract

- Input buffer: 512 `uint32` words, 2048 bytes.
- Output buffer: 40 `uint32` words, 160 bytes.
- Start S2MM receive before MM2S transmit.
- Input `TLAST` is generated by AXI DMA at byte 2047.
- Output `TLAST` is generated by the decoder on word 39.
- Input and output `TKEEP` are full-beat `4'hf` on every word.

## First Debug Checks

If the receive DMA hangs, check:

- `aresetn` is high after the PS clock is stable.
- S2MM was started before MM2S.
- Decoder output asserts `m_axis_tlast` on word 39.
- Output `m_axis_tready` from the DMA is high during the response.

If an error response appears, check:

- MM2S length is exactly 2048 bytes.
- All input beats have `TKEEP=4'hf`.
- `TLAST` appears only on input word 511.

If status passes but bits do not match, check:

- Four signed int8 LLRs are packed little-lane into each 32-bit word.
- Decoded bit `i` is bit `i % 32` of output word `8 + floor(i / 32)`.
